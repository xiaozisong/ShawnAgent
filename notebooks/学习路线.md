### Day 1 — RAG 基础与文档处理

| 项目 | 内容 |
|------|------|
| **学习目标** | 理解 RAG 全流程：文档加载 → 文本分块 → 向量化 → 检索 → 生成 |
| **学习内容** | 文档解析、语义分块、Embedding、向量检索、检索重排 |
| **对应章节** | [第八章 记忆与检索 - RAG 部分](../hello-agents/docs/chapter8/第八章%20记忆与检索.md) |

**RAG Pipeline 示意图：**

```
文档 → [加载/解析] → [语义分块] → [生成 Embedding] → [存入向量库]
                                                          ↓
用户提问 → [向量化] → [语义检索] → [检索结果] ─┐
                                              ↓
                                          [LLM 生成回答] ← 拼接上下文
```

**关键概念：**
- 语义分块：按段落/章节边界切分，保持语义完整
- Embedding：文本 → 高维向量（语义相近则向量距离近）
- 向量检索：余弦相似度计算
- 检索重排 (Re-ranking)：用 Cross-Encoder 精排粗筛结果

**动手练习：**

### Day 3 — 上下文工程

| 项目 | 内容 |
|------|------|
| **学习目标** | 解决 Agent 在长时程任务中的"上下文腐蚀"问题 |
| **学习内容** | GSSC 流水线（Gather-Select-Structure-Compress） |
| **对应章节** | [第九章 上下文工程](../hello-agents/docs/chapter9/第九章%20上下文工程.md) |

**GSSC 上下文构造流水线：**

```
Gather（汇集） → Select（筛选） → Structure（结构化） → Compress（压缩）
   │                │                  │                   │
多源信息收集    相关度评分        6分区模板         Token 预算控制
               + 新近性排序                        + 摘要压缩
```

**关键概念：**
- 上下文腐蚀：Agent 在长流程中逐渐"忘记"早期信息
- JIT 即时上下文检索：必要时才检索，不把所有历史塞入提示
- 分层策略：压缩整合 / 结构化笔记 / 子代理架构

**MCP 架构：**

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  Host    │───▶│  Client  │───▶│  Server  │
│ (Agent)  │    │ (SDK)    │    │ (Tool)   │
└──────────┘    └──────────┘    └──────────┘
     │              │                │
  对话界面       发现工具         实际执行
  与上下文     调用工具         返回结果
     │              │                │
  ┌──────────────┴────────────────────────┐
  │  传输：Stdio / HTTP / SSE / StreamableHTTP │
  └──────────────────────────────────────────┘
```
