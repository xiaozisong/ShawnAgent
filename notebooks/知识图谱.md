### 2.2 Agent 核心层

```
┌─────────────────────────────────────────────────────────────────┐
│  Agent 核心层                                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【Agent 基础概念】                                             │
│  ├─ 定义：感知 → 思考 → 行动的自主实体                         │
│  ├─ PEAS 模型：P(性能) E(环境) A(执行器) S(传感器)             │
│  ├─ Agent Loop：感知 → 思考(规划+工具选择) → 行动 → 观察       │
│  ├─ Workflow vs Agent：预设流程 vs 自主决策                    │
│  └─ 分类维度                                                   │
│      ├─ 传统：反射式 → 模型式 → 目标式 → 效用式 → 学习式      │
│      ├─ 时间：反应式 vs 规划式 vs 混合式                       │
│      └─ 知识：符号主义 vs 亚符号主义 vs 神经符号主义           │
│                                                                 │
│  【三种经典范式】                                              │
│  ├─ ReAct (Reasoning + Acting)                                 │
│  │   核心：Thought → Action → Observation 循环迭代             │
│  │   适用：需要外部工具和动态纠错的场景                         │
│  │   代码：hello-agents/code/chapter4/                          │
│  │                                                             │
│  ├─ Plan-and-Solve                                            │
│  │   核心：先制定完整计划 → 逐步执行计划                       │
│  │   适用：需要多步推理的结构化任务                             │
│  │   代码：hello-agents/code/chapter4/                          │
│  │                                                             │
│  └─ Reflection                                                 │
│      核心：执行 → 自我反思 → 修正 → 重新执行                   │
│      适用：对输出质量有极致要求的场景（代码生成、报告）         │
│      代码：hello-agents/code/chapter4/                          │
│                                                                 │
│  【Agent 基本组件】                                            │
│  ├─ LLM 客户端（模型调用抽象层）                                │
│  ├─ Prompt 模板（系统提示词 + 任务指令）                       │
│  ├─ 输出解析器（正则 / AST 解析 Thought-Action）               │
│  ├─ 工具系统（注册 / 调用 / 结果处理）                         │
│  ├─ 记忆系统（工作记忆 / 情景记忆 / 语义记忆）                 │
│  └─ 主循环控制器（循环步数 / 终止条件 / 错误处理）            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

### 2.6 训练优化层

```
┌─────────────────────────────────────────────────────────────────┐
│  训练优化层                                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【Agentic RL（智能体强化学习）】                               │
│  ├─ 训练流程                                                    │
│      ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│      │ 数据准备 ├───▶│ SFT 训练 ├───▶│GRPO 训练 │              │
│      └──────────┘    └──────────┘    └────┬─────┘              │
│                                            ▼                    │
│                                     ┌──────────┐               │
│                                     │ 评估/部署 │               │
│                                     └──────────┘               │
│                                                             │
│  ├─ SFT（监督微调）                                           │
│  │   ├─ 目的：让模型学会输出格式（Thought-Action 格式）        │
│  │   ├─ 数据格式：{"prompt":"...", "chosen":"..."}            │
│  │   └─ 使用 LoRA（低秩适配）减少训练参数量                    │
│  │                                                             │
│  ├─ GRPO（群组相对策略优化）                                  │
│  │   ├─ 目的：通过强化学习让 Agent 学会更好的策略              │
│  │   ├─ 核心：组内生成多个回答 → 相对奖励 → 策略更新         │
│  │   ├─ 奖励函数：准确率 + 长度惩罚 + 步骤奖励                 │

## 四、关键概念对照表

### 4.1 前端开发者 → Agent 开发概念映射

| 前端概念 | Agent 开发概念 | 说明 |
|---------|--------------|------|
| Redux Store | LangGraph State | 全局状态对象，可被任何节点读取/写入 |
| Redux Reducer | Graph Node | 接收当前 state，处理并返回新 state |
| React Router | Conditional Edge | 根据条件选择下一个路由/节点 |
| Promise / async/await | Agent 异步循环 | 串行/并行执行多个步骤 |
| Middleware | Agent 工具 | 可插拔的功能扩展点 |
| REST API | Tool 函数 | 输入参数 → 执行 → 返回结果 |
| WebSocket / SSE | Agent 流式输出 | 实时推送更新 |
| Component 组合 | Agent 嵌套 | 子 Agent 被父 Agent 调用 |
| useEffect | Reflection 范式 | 响应变化并执行副作用 |
| TypeScript interface | Pydantic BaseModel | 运行时数据验证 + 类型约束 |

### 4.2 数据流形态对比

```
传统 Web App:
  用户输入 → REST API → 后端处理 → 数据库 → 返回 JSON → 渲染 UI
           ↑                                                    │
           └──────────────── 请求-响应 ──────────────────────────┘

Agent 应用:
  用户输入 → LLM 推理 → 调用工具 → 观察结果 → LLM 再次推理 → ... → 最终输出
            ↑                                                    │
            └────────────── Agent Loop（多轮迭代） ────────────────┘
```

### 4.3 各框架 Tool 注册对比

| 框架 | 工具注册方式 | 示例 |
|------|------------|------|
| LangChain | `@tool` 装饰器 | `@tool def my_tool(): ...` |
| LangGraph | 作为 Node 调用 Tool | `tools_node = ToolNode([...])` |
| CrewAI | Agent 的 tools 参数 | `Agent(tools=[...])` |
| AutoGen | 注册到 UserProxyAgent | `UserProxyAgent(function_map={...})` |
| HelloAgents | ToolRegistry 注册 | `tool_registry.register(...)` |

### 4.4 Agent 架构选型矩阵

| 需求 | 推荐技术选型 |
|------|------------|
| 快速原型验证 | Coze / Dify（低代码） |
| 简单单 Agent | LangChain + ChatOpenAI |
| 复杂工作流控制 | LangGraph（StateGraph） |
| 多 Agent 团队协作 | CrewAI / AutoGen |
| 生产级高并发 | AgentScope / LangGraph |
| 教学 / 理解原理 | HelloAgents（自研） |
| 需要 RAG | LangChain + Chroma / Qdrant |
| 全栈 Web 应用 | FastAPI + Vue3 + LangGraph |
| 训练自定义模型 | TRL + Qwen + GRPO |
